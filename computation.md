# PoliePuter — physical / optical computation

*A use of the [Reality Kernel](reality_kernel/) as a physical computer. Patent pending; all rights
reserved.*

The PoliePuter treats the Reality Kernel's physical channel as a **computational substrate**: an optical medium —
resonance chambers, nonlinear or epsilon-near-zero media — in which the kernel can realise operations such as
**optical matrix multiplication, 2D correlation, and Fourier / fractional-Fourier transforms** directly in the
physics (a linear medium suffices for these primitives), as described and enabled in the filings. Learning agents
can be bootstrapped on the device's own emission→response data to find useful operating points.

This is the broadest extension of the kernel into physical computation: its **performance and hardness
envelope** is the part of the suite still being characterised — these physical-computation embodiments are
described and enabled in the filings, with their envelope still being mapped. It uses the same
**convolution-bundle** data model, meters, and protocol digest as the other regimes. See [`reality_kernel/`](reality_kernel/) for the authoritative description.
